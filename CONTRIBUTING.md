# Contributing to avvarre

## Setup

```bash
git clone https://github.com/PralhadYadawad/avvarre.git
cd avvarre
npm install && npm run build
```

Run tests: `node dist/test-rules.js` or specific language runners (e.g., `node dist/test-html-rules.js`).

## Project Structure

| Layer | Path | What It Does |
|-------|------|-------------|
| Rules | `src/analyzer/rules/` | 430+ regex rules across 21 languages |
| Engine | `src/analyzer/engine.ts` | Language dispatch + rule execution |
| Scorer | `src/analyzer/scorer.ts` | 0-100 scoring + A-F grading |
| AI | `src/ai/` | Gemini + OpenAI-compatible clients, prompts, chunker |
| Ecosystem | `src/ecosystem/` | Scaffold, skills, history, badges, masking |
| Workspace | `src/workspace/` | Directory scanning, PR quality gate |
| MCP Server | `src/server.ts` | 9 tools + 21 resources |
| Plugin | `plugin/` | VS Code agent plugin (hooks, commands, agent, skills) |

## Adding a New Language

1. Read the full Google Style Guide for the target language
2. Create `src/analyzer/rules/<language>.ts` with rules grouped by style guide section
3. Implement `getCleanLines` — strips comments and strings to prevent false positives
4. Register in `src/analyzer/engine.ts` (language dispatch) and `src/server.ts` (Zod enums)
5. Add AI prompt in `src/ai/prompts.ts`
6. Create test file: `codebase_test/bad_code.<ext>` with intentional violations
7. Create test runner: `src/test-<language>-rules.ts`
8. Update language count in documentation if total changes

Rule ID format: `LANG-CATEGORY-NN` (e.g., `PY-EXC-01`, `TS-TYPE-03`).

## Adding a Slash Command

1. Create `plugin/commands/<name>.md` with YAML frontmatter (`title`, `description`)
2. Write the prompt body — reference MCP tools by name (e.g., `avvarre_file`, `avvarre_workspace`)
3. Test locally by enabling the plugin in VS Code 1.110+

## Adding a Hook

1. Add an entry to `plugin/hooks.json` — fields: `event`, `description`, `condition`, `handler`
2. Events: `onAgentStart`, `onAgentEnd`, `onFileSave`
3. Keep handlers to 1-2 sentences. Hooks are prompt injections — shorter = more reliable
4. Never make hooks auto-destructive (no auto-deleting, no force-pushing)

## Adding a Skill Template

Skills are generated by `src/ecosystem/skill_generator.ts` based on detected tech stack.

1. Add detection logic in `generateSkills()` — check `techStack` or `keyFeatures`
2. Write a skill template as a markdown string with architectural rules
3. Skills should be specific (e.g., "React UI Guidelines") not generic ("How to Code")

Community skills are fetched by `src/ecosystem/community_fetcher.ts` from [awesome-cursorrules](https://github.com/PatrickJS/awesome-cursorrules).

## Conventions

- TypeScript strict mode, ES modules, `.js` extensions in imports
- `camelCase` functions, `PascalCase` types, `UPPER_SNAKE_CASE` constants, `snake_case` filenames
- Max 1500 lines per file
- Every exported function has a JSDoc docstring
- See `.avvarre/conventions.md` for the full list

## The Compressed Task Protocol

When updating `.avvarre/tasks.md`, use compressed format:

```
[x] Summary (steps: a→b→c→d→e)
[/] Partial (done: a→b | next: c→d)
[ ] New task
```

3-5 internal subtasks = 1 line. The step chain is the expansion key for the next contributor.

## Pull Requests

1. Run `/avvarre:pr` or use the `avvarre_pr` MCP tool before submitting
2. All PRs target the `master` branch
3. Include test files for new rules
4. Update `.avvarre/tasks.md` if your change completes or adds tasks
